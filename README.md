# TCP 可靠数据传输协议模拟器

这是一个用 JavaFX 写的计算机网络作业/小工具，主要是为了动态、直观地演示 TCP 协议中**滑动窗口**和**可靠传输**的核心机制。

> **_本项目为 Vibe Coding 产物_**

## 实现了哪些机制？

相比于简单的停等协议，这个模拟器主要展示了 TCP 的以下几个精髓：

- **滑动窗口 (Sliding Window)**：动态显示发送方的窗口限制。底部有一条履带式的序列号跑道，能直观看到窗口高亮层是如何随着 ACK 的到来往前滑动。
- **累积确认 (Cumulative ACK)**：TCP 不会为每一个包单独发确认。如果中途有个 ACK 丢失，但后面的 ACK 成功到达，发送方也会明白前面的包已经安全送达，窗口向前滑动。
- **超时重传 (Timeout Retransmission)**：每个发出去的包都有定时器。如果包在半路丢失，等待超时后，会自动补发重传包。
- **动态窗口 & 批量发送**：可随时在界面上把窗口调大调小（当然，不能把正在飞的包给切掉），也可以一次性发一串包，看它们排队飞过去。

## 环境要求

- **JDK**: 21
- **构建**: Maven

## 怎么跑起来？

克隆或下载代码，在项目根目录打开终端：

```bash
mvn clean compile javafx:run
```

## 怎么玩？

启动后，可以尝试这几种典型场景：

1.  **正常连发**：把“批量发送数量”设成 3，点发送。观察包排队飞跃和窗口平滑移动。
2.  **把窗口塞满**：窗口大小设为 4，一次性要求发 6 个包。多出来的请求会被拒绝，体现了流量控制。
3.  **丢失数据包**：点“发送1个并模拟数据丢失”。包飞到一半丢失，接收方没反应。定时器超时，黄色的包自动飞出。
4.  **丢失 ACK（测试累积确认）**：
    - 点击“发送并模拟 ACK 丢失”（包 0 过去，ACK 0 回来时丢失）。
    - 马上点普通的“发送”发包 1（包 1 过去，ACK 1 成功回来）。
    - 发送方收到 ACK 1 后，直接把窗口滑动到了包 2 的位置，并取消了包 0 的重传（因为它知道包 0 其实对方收到了）。

## 📁 代码结构说明

- `TcpSimulator.java`：业务逻辑。维护窗口大小、base 序号、nextSeqNum，运行程序主逻辑。
- `TcpSimulatorUI.java`：图像绘制。绘制所有的按钮、动画 (TranslateTransition)等 UI。
- `App.java`：入口文件。
